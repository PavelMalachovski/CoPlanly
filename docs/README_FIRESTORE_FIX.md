# 🔧 Исправление ошибки "Permission denied" в Firestore

## ⚠️ Проблема
Вы видите ошибку: *"Permission denied. Please check Firestore security rules..."*

## ✅ Решение (пошагово)

### Шаг 1: Разверните правила безопасности Firestore

**Вариант A: Используйте упрощенные правила (для быстрого тестирования)**

1. Откройте файл `firestore.rules.simple` в корне проекта
2. Скопируйте **все** содержимое файла
3. Откройте Firebase Console: https://console.firebase.google.com/project/coparently-a39c9/firestore/rules
4. Вставьте содержимое в редактор правил
5. Нажмите кнопку **"Publish"** (Опубликовать)
6. Дождитесь подтверждения

**Вариант B: Используйте полные правила (для продакшена)**

1. Откройте файл `firestore.rules` в корне проекта
2. Скопируйте **все** содержимое файла
3. Откройте Firebase Console: https://console.firebase.google.com/project/coparently-a39c9/firestore/rules
4. Вставьте содержимое в редактор правил
5. Нажмите кнопку **"Publish"** (Опубликовать)
6. Дождитесь подтверждения

### Шаг 2: Перезапустите приложение

1. **Полностью закройте приложение** (не просто сверните)
2. Или удалите приложение с устройства и установите заново
3. Или очистите данные приложения:
   - Settings → Apps → CoParently → Storage → Clear Data

### Шаг 3: Проверьте

1. Запустите приложение заново
2. Войдите в систему
3. Попробуйте выполнить "co-parent pairing"

## 🔍 Диагностика

### Если ошибка сохраняется:

1. **Откройте Logcat в Android Studio:**
   - Вкладка "Logcat" внизу
   - Фильтр: `PairingViewModel` или `FirestoreUserDataSource`
   - Найдите сообщения об ошибках

2. **Проверьте аутентификацию:**
   - Убедитесь, что вы действительно вошли в систему
   - Попробуйте выйти и войти заново

3. **Проверьте статус API:**
   - Откройте: https://console.developers.google.com/apis/api/firestore.googleapis.com/overview?project=coparently-a39c9
   - Убедитесь, что API имеет статус **"Enabled"**

4. **Проверьте базу данных:**
   - Откройте: https://console.firebase.google.com/project/coparently-a39c9/firestore
   - Убедитесь, что база данных создана

## 📝 Важные моменты

- ✅ Правила безопасности **ОБЯЗАТЕЛЬНО** должны быть развернуты в Firebase Console
- ✅ После развертывания правил может потребоваться несколько минут, чтобы они вступили в силу
- ✅ Если упрощенные правила работают, но полные нет - проблема в сложности правил
- ✅ Если упрощенные правила НЕ работают - проблема не в правилах, а в другом (аутентификация, API, и т.д.)

## 🚀 Быстрая проверка через Firebase CLI

Если у вас установлен Firebase CLI:

```bash
# Развернуть правила
firebase deploy --only firestore:rules

# Проверить статус
firebase projects:list
```
